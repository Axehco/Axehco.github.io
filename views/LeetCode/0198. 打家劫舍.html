# 198. 打家劫舍

钟 波

# 题目描述

题目传送门：198. 打家劫舍 (opens new window)

你是一个专业的小偷，计划偷窃沿街的房屋。每间房内都藏有一定的现金，影响你偷窃的唯一制约因素就是相邻的房屋装有相互连通的防盗系统，如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警。

给定一个代表每个房屋存放金额的非负整数数组，计算你 不触动警报装置的情况下 ，一夜之内能够偷窃到的最高金额。

示例 1：

输入：[1,2,3,1]
输出：4
解释：偷窃 1 号房屋 (金额 = 1) ，然后偷窃 3 号房屋 (金额 = 3)。
     偷窃到的最高金额 = 1 + 3 = 4 。

示例 2：

输入：[2,7,9,3,1]
输出：12
解释：偷窃 1 号房屋 (金额 = 2), 偷窃 3 号房屋 (金额 = 9)，接着偷窃 5 号房屋 (金额 = 1)。
     偷窃到的最高金额 = 2 + 9 + 1 = 12 。

提示：

  • 1 <= nums.length <= 100
  • 0 <= nums[i] <= 400

# 题解

经典的动态规划问题。

1、定义dp数组

dp[i] ：前i个房子在满足条件下的能偷窃到的最高金额为dp[i] 。

2、确定状态转移方程

关键就在于第i间房我们偷还是不偷：

  • 如果偷第i间房，那我们就不能偷第i-1间，此时的最高总金额为前i-2间的最高总金额与第i间房的金额之和。
  • 如果不偷第i间房，此时的总金额就是前i-1间房能偷到的最高总金额。

于是得到状态转移方程为：

3、边界条件

只有一间房时，就只能偷该间房。有两间房时选择金额较大的一间进行偷窃。

dp[0] = nums[0];
dp[1] = Math.max(nums[0], nums[1]);

4、确定dp数组的计算顺序

已知前两个状态，可推出第三个状态，因此从前往后计算。

代码如下：

class Solution {
    public int rob(int[] nums) {
        if (nums.length == 1) return nums[0];
        int[] dp = new int[nums.length];
        dp[0] = nums[0];
        dp[1] = Math.max(nums[0], nums[1]);
        for (int i = 2; i < nums.length; i++) {
            dp[i] = Math.max(dp[i - 2] + nums[i], dp[i - 1]);
        }
        return dp[nums.length - 1];
    }
}

我们发现dp[i]只与dp[i-1]和dp[i-2]有关，因此不用存储整个dp数组，直接用两个变量来存储即可。

代码如下：

class Solution {
    public int rob(int[] nums) {
        if (nums.length == 1) return nums[0];
        int pre = nums[0];
        int cur = Math.max(nums[0], nums[1]);
        for (int i = 2; i < nums.length; i++) {
            int tempMax = Math.max(cur, pre + nums[i]);
            pre = cur;
            cur = tempMax;
        }
        return cur;
    }
}

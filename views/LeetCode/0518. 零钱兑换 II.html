# 518. 零钱兑换 II

钟 波

# 题目描述

题目传送门：518. 零钱兑换 II (opens new window)

给你一个整数数组 coins 表示不同面额的硬币，另给一个整数 amount 表示总金额。

请你计算并返回可以凑成总金额的硬币组合数。如果任何硬币组合都无法凑出总金额，返回 0 。

假设每一种面额的硬币有无限个。

题目数据保证结果符合 32 位带符号整数。

示例 1：

输入：amount = 5, coins = [1, 2, 5]
输出：4
解释：有四种方式可以凑成总金额：
5=5
5=2+2+1
5=2+1+1+1
5=1+1+1+1+1

示例 2：

输入：amount = 3, coins = [2]
输出：0
解释：只用面额 2 的硬币不能凑成总金额 3 。

示例 3：

输入：amount = 10, coins = [10] 
输出：1

# 题解

本题假设每一种面额的硬币有无限个，给出了凑要凑出的总金额，即背包容量，典型的完全背包问题。

但是和纯完全背包问题不太一样，本题是需要计算的是选取硬币的 组合 数。因此不考虑元素之间的顺序。

1、定义dp数组

dp[j] ： 凑满容量为j的金额，有dp[j]种方法。

2、确定状态转移方程

不考虑coins[i]的情况下，填满容量为j - coins[i]的背包，有dp[j - coins[i]]种方法。

也就是当前填满容量为j的背包的方法数 = 之前填满容量为j的背包的方法数 + 之前填满容量为j - coins[i]的方法数。

于是得到状态转移方程为：

3、边界条件

当背包容量为0时，即不选取任何硬币时，金额之和才为 0 ，因此只有1种硬币组合。

dp[0] = 1;

4、确定dp数组的计算顺序

一维dp数组解决完全背包问题，先正序遍历物品，再正序遍历背包。

代码如下：

class Solution {
    public int change(int amount, int[] coins) {
        int[] dp = new int[amount + 1];
        dp[0] = 1;
        for (int i = 0; i < coins.length; i++) {
            for (int j = coins[i]; j < amount + 1; j++) {
                dp[j] = dp[j] + dp[j - coins[i]];
            }
        }
        return dp[amount];
    }
}

上面的解法已经是状态压缩后的优化写法，当然也可以用二维dp来解决。写法框架可以参考 零壹背包基础

代码如下：

class Solution {
    public int change(int amount, int[] coins) {
        // 定义dp： dp[i][j]表示使用前i个物品, 恰好装满容量为j的背包的方法个数
        // 即：下标为[0,i]的coins[i]能够凑满容量为j的背包，有dp[i][j]种方法。
        int[][] dp = new int[coins.length + 1][amount + 1];
        
        // 边界条件：当背包容量为0时，即不选取任何硬币时，金额之和才为0 ，因此只有1种硬币组合。
        dp[0][0] = 1;
        
        // dp数组计算顺序：先遍历物品、再遍历背包 【求组合】
        for (int i = 1; i < coins.length + 1; i++) {
            // 注意这里从0开始
            for (int j = 0; j < amount + 1; j++) {
                if (j < coins[i - 1]) {  // 容量不够不能装入背包
                    dp[i][j] = dp[i -1][j];
                } else {  // 能装入背包，但选择 装 与 不装
                    // 当前的方案数应该是装入背包与不装入背包的总和
                    dp[i][j] = dp[i - 1][j] + dp[i][j - coins[i - 1]];
                }
            }
        }
        return dp[coins.length][amount];
    }
}

代码虽然过了，但是有一点需要注意，针对本题： 我们不关心硬币使用的顺序，而是硬币有没有被用到

本题可以这样想：

problem(k,i) = problem(k-1, i) + problem(k, i-coins[k])

即前k个硬币凑齐金额i的组合数 等于 前k-1个硬币凑齐金额i的组合数 加上 在原来i-k的基础上使用硬币的组合数。 说的更加直白一点，就是用前k的硬币凑齐金额i，要分为两种情况计算， 一种是没有用第k个硬币就凑齐了，一种是前面已经凑到了i-coins[k]，现在就差第k个硬币了。

因此外层循环先遍历coins ，也就是枚举硬币，类比背包问题中的物品，再枚举金额，类比背包问题中的背包。先把硬币顺序给固定了。

如果是考虑排列数，即有顺序地进行排列，那就需要先遍历背包、再遍历物品了 。 后面这种求排列数的方法类似于 70. 爬楼梯 的思想，爬楼梯这道题的本质其实就是排列。

总结：

如果求组合数就是外层for循环遍历物品，内层for遍历背包。

如果求排列数就是外层for遍历背包，内层for循环遍历物品。

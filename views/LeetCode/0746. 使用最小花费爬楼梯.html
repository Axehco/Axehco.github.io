# 746. 使用最小花费爬楼梯

钟 波

# 题目描述

题目传送门：746. 使用最小花费爬楼梯 (opens new window)

给你一个整数数组 cost ，其中 cost[i] 是从楼梯第 i 个台阶向上爬需要支付的费用。一旦你支付此费用，即可选择向上爬一个或者两个台阶。

你可以选择从下标为 0 或下标为 1 的台阶开始爬楼梯。

请你计算并返回达到楼梯顶部的最低花费。

示例 1：

输入：cost = [10,15,20]
输出：15
解释：你将从下标为 1 的台阶开始。
- 支付 15 ，向上爬两个台阶，到达楼梯顶部。
总花费为 15 。

示例 2：

输入：cost = [1,100,1,1,1,100,1,1,100,1]
输出：6
解释：你将从下标为 0 的台阶开始。
- 支付 1 ，向上爬两个台阶，到达下标为 2 的台阶。
- 支付 1 ，向上爬两个台阶，到达下标为 4 的台阶。
- 支付 1 ，向上爬两个台阶，到达下标为 6 的台阶。
- 支付 1 ，向上爬一个台阶，到达下标为 7 的台阶。
- 支付 1 ，向上爬两个台阶，到达下标为 9 的台阶。
- 支付 1 ，向上爬一个台阶，到达楼梯顶部。
总花费为 6 。

提示：

  • 2 <= cost.length <= 1000
  • 0 <= cost[i] <= 999

# 题解

题目要求的是 到达 第n级台阶楼层 顶部 的最小花费，可以用动态规划来解。

1、定义dp数组

离开第i个台阶时的最低花费， 注意 ：是离开，这点很重要，不是还未到达。

2、确定状态转移方程

对某个台阶i来说，我们可以从的台阶i-1过来，也可以从台阶i-2过来，那具体从哪儿过来，取决于到那儿花费的体力小，这样才能使得我们最终的答案整体更小。因此dp[i]取决于dp[i-1]和dp[i-2]的大小，哪个小我们取哪个，再加上离开本台阶时需要的花费cost[i]即可。于是得到状态转移方程为：

3、边界条件

dp[0] = cost[0];
dp[1] = cost[1];

4、确定dp数组的计算顺序

已知前两个状态的最小花费，我们可推出第三个状态的信息，因此从前往后计算。

本题的到达终点，我们应该从倒数第一个台阶过来，或者倒数第二个台阶过来，就看谁的花费小，我们就从谁哪儿过来，所以最后要取 dp数组 的最后两位的最小值作为答案，才是最终到达楼梯顶部的最小花费。

代码如下：

class Solution {
    public int minCostClimbingStairs(int[] cost) {
        int[] dp = new int[cost.length];
        dp[0] = cost[0];
        dp[1] = cost[1];
        for (int i = 2; i < cost.length; i++) {
            dp[i] = Math.min(dp[i - 1], dp[i - 2]) + cost[i];
        }
        return Math.min(dp[dp.length - 1], dp[dp.length - 2]);
    }
}

注意到状态转移方程中只用到了前面的两个记录，可以不用记录整个一维数组，只用两个变量保存前面的两个记录，并不断更新即可，这样可降低空间复杂度。

优化的代码如下：

class Solution {
    public int minCostClimbingStairs(int[] cost) {
        int pre = cost[0], cur = cost[1];
        for (int i = 2; i < cost.length; i++) {
            int minCost = Math.min(pre, cur) + cost[i];
            pre = cur;
            cur = minCost;
        }
        return Math.min(pre, cur);
    }
}

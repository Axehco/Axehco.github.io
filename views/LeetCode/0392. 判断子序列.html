# 392. 判断子序列

钟 波

# 题目描述

题目传送门：392. 判断子序列 (opens new window)

给定字符串 s 和 t ，判断 s 是否为 t 的子序列。

字符串的一个子序列是原始字符串删除一些（也可以不删除）字符而不改变剩余字符相对位置形成的新字符串。（例如，"ace"是"abcde"的一个子序列，而"aec"不是）。

进阶：

如果有大量输入的 S，称作 S1, S2, ... , Sk 其中 k >= 10亿，你需要依次检查它们是否为 T 的子序列。在这种情况下，你会怎样改变代码？

示例 1：

输入：s = "abc", t = "ahbgdc"
输出：true

示例 2：

输入：s = "axc", t = "ahbgdc"
输出：false

提示：

  • 0 <= s.length <= 100
  • 0 <= t.length <= 10^4
  • 两个字符串都只由小写字符组成。

# 题解

# 双指针

定义双指针i和j，分别指向字符串s和t。

遍历字符串t，边遍历边比较即可。

注意空串""都是字符串t的字串，因为题目说的可以删除字符，将t全部删除就变为空串了。

代码如下：

class Solution {
    public boolean isSubsequence(String s, String t) {
        int i = 0, j = 0;
        while (i < s.length() && j < t.length()) {
            if (s.charAt(i) == t.charAt(j)) {
                i++;
            }
            j++;
        }
        return i == s.length();
    }
}

# 动态规划

本题使用动态规划解决需要转一个弯，在字符串t中，遇到两个字符串不匹配时，继续查找下一个，类似于删除当前字符（往下看就明白了）。

1、定义dp数组

dp[i][i]表示以[0:i-1]为下标构成的字符串s，和以[0:j-1]为下标构成的字符串t，其相同子序列的长度为dp[i][j] 。

2、确定状态转移方程

当遍历到字符串t中的字符也在字符串s中出现时，就找到一个相同的字符，此时以[0:i-1]和[0:j-1]为下标构成的相同子序列的长度为： dp[i][j] = dp[i-1][j-1] + 1

当遍历到字符串t中的字符不在字符串s中出现时，此时就在t中模拟删除当前字符，继续匹配，也就是说当前删除的字符是t[j-1]，所以此时的dp[i][j]的结果其实就是dp[i][j-1] ，也就是之前s[i-1]与t[j-2]比较的结果，这点需要好好理解。

3、边界条件

定义dp数组的时候就多初始化了一排空间，dp[i][0]实际上没有意义，只是为了后序的结果推出而已，初始化为0即可。

4、确定dp数组的计算顺序

dp[i][j]由dp[i][j-1]和dp[i-1][j-1]推出，因此外循环先遍历s，内循环再遍历t即可。

代码如下：

class Solution {
    public boolean isSubsequence(String s, String t) {
        int[][] dp = new int[s.length() + 1][t.length() + 1];
        for (int i = 1; i < s.length() + 1; i++) {
            for (int j = 1; j < t.length() + 1; j++) {
                if (s.charAt(i - 1) == t.charAt(j - 1)) {
                    dp[i][j] = dp[i - 1][j - 1] + 1;
                } else {
                    dp[i][j] = dp[i][j - 1];
                }
            }
        }
        return dp[s.length()][t.length()] == s.length();
    }
}

注意结果的返回，就是就是看dp数组的最后一位是不是字符串s的长度。还有一点是本题的字符串t是远远长于s的，这点需要注意。
